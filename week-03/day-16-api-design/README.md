---

## Learning Objectives
- Design RESTful APIs following best practices
- Explain gRPC and when to use it over REST
- Design API versioning strategies
- Explain API gateway patterns
- Design pagination, filtering, and sorting for APIs

---

## Quick Summary

An API is a contract. Once you publish it, clients depend on it. Breaking changes break clients. Good API design is about creating contracts that are clear, consistent, and evolvable.

The core insight: **API design is permanent. Bad APIs are forever. Invest time in getting it right before you publish.**
