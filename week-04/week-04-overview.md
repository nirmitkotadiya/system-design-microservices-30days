# Week 4: Synthesis — Real Systems, Real Interviews

## Focus Area
Putting it all together. This week simulates real system design interviews using actual systems you use every day. By the end, you'll have a repeatable framework for approaching any system design problem.

## Prerequisites
- Weeks 1–3 complete
- All concepts from previous weeks internalized
- Ability to sketch architectures quickly

## Learning Objectives
By the end of Week 4, you will be able to:
- Design Twitter, Uber, Netflix, and WhatsApp from scratch
- Apply security best practices to distributed systems
- Use a structured framework for any whiteboard design problem
- Answer system design interview questions at senior engineer level
- Identify your knowledge gaps and create a personal study plan

## Required Tools
- Whiteboard or paper (seriously)
- Timer (for mock interviews)
- All tools from previous weeks

## Success Criteria
You've mastered Week 4 when you can:
- [ ] Design any of the four case study systems in 45 minutes
- [ ] Explain the security implications of your designs
- [ ] Use a structured framework without prompting
- [ ] Identify the bottleneck in any system you design
- [ ] Discuss tradeoffs confidently without memorized answers
