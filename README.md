## 🎯 What Is This?

This repository is a structured, self-paced learning path for **system design** — the skill that separates mid-level engineers from senior engineers, and senior engineers from staff engineers.

By the end of 30 days, you will be able to:
- Design scalable distributed systems from scratch
- Reason about tradeoffs like a senior engineer
- Ace system design interviews at top tech companies
- Understand how real systems (Twitter, Uber, Netflix) actually work

**Primary Reference**: [Designing Data-Intensive Applications](https://dataintensive.net/) by Martin Kleppmann (referred to as "the Bible" throughout this curriculum)

## ✅ Prerequisites

Before starting, you should have:

- [ ] Basic programming experience (any language — Python preferred for examples)
- [ ] Familiarity with HTTP and REST APIs
- [ ] Basic understanding of what a database is
- [ ] Ability to run Docker containers locally
- [ ] 1–2 hours per day to dedicate to learning

You do NOT need:
- Prior distributed systems experience
- A CS degree
- Experience at a large tech company

## 🚀 How to Use This Repository

### Step 1: Fork and Clone
```bash
git clone https://github.com/nirmitkotadiya/system-design-microservices-30days.git
cd system-design-microservices-30days
```

### Step 2: Set Up Your Environment
```bash
# Install Docker (required for code examples)
# https://docs.docker.com/get-docker/

# Install Python 3.9+
python --version

# Install dependencies for code examples
pip install -r requirements.txt
```

### Step 3: Follow the Daily Structure
Each day folder contains:
1. `README.md` — Start here. Overview and objectives.
2. `concepts.md` — Deep dive into the theory.
3. `exercises.md` — Hands-on practice.
4. `code-examples/` — Runnable code.
5. `diagrams/` — Visual aids.

### Step 4: Track Your Progress
Use the progress table below. Check off each day as you complete it.

### Step 5: Do the Exercises
Don't skip exercises. The hands-on work is where the learning happens.

### Step 6: Join the Community
Share your progress, ask questions, and help others (see Community section).
